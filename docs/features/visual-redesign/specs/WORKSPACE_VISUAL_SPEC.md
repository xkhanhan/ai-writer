# 书籍工作区前端视觉改造设计文档

## 概述

对书籍创作工作区进行全面视觉改造，建立统一的「东方水墨风格」设计体系，重构 8 个面板（含 ActivityBar 导航），规范全局交互模式。

视觉稿：`.superpowers/brainstorm/15020-1783147932/content/design-final.html`

## 设计原则

1. **内容简约**：主区域不堆叠过多按钮/标签，复杂操作用折叠、下拉、弹窗收拢
2. **AI 统一入口**：所有 AI 功能收拢在一个 `AI ▾` 下拉按钮中，根据上下文动态控制可用状态
3. **创作区纯编辑**：创作区不存在观看模式，全部为编辑状态，右上角固定「保存」按钮，手动保存
4. **面包屑导航**：所有面板使用 antd 面包屑 + Outlined 图标，替代面板标题
5. **删除二次确认**：所有删除操作使用 antd Modal.confirm

## 全局交互规范

### AI 下拉按钮

- 按钮样式：`AI ▾`（描边按钮 + 火花图标 + 下拉箭头）
- 菜单项按顺序：`生成内容` | ---分隔线--- | `去除 AI 味` | `润色` | `扩写`
- 条件控制：
  - 「生成内容」— 始终可用
  - 「去除AI味」— 仅 AI 生成后可用
  - 「润色」「扩写」— 仅有内容时可用
  - 不可用时菜单项灰置（`color: #ccc`），不隐藏

### 面包屑导航

- 格式：`{icon} 《书名》 › {面板名} › {列表} › {详情}`
- 使用 antd `Breadcrumb` 组件 + `@ant-design/icons` Outlined 系列图标

### 删除确认弹窗

- antd `Modal.confirm`
- 标题：「确认删除」
- 内容：「确定要删除「{名称}」吗？此操作不可撤销。」
- 确认按钮 `danger`，取消按钮默认

### 空状态

- 统一风格：32px 灰色 SVG 图标（`stroke-width: 1`）+ 提示文字 + 引导按钮
- 引导按钮使用虚线边框 `border: 1px dashed var(--color-primary)`

### 创作区保存机制

- 创作区全部为编辑模式，无观看模式
- 右上角固定显示「保存」按钮（描边按钮 + 保存图标）
- 不支持自动保存
- 保存状态在底部状态栏显示

## 面板设计

### 0. ActivityBar 面板导航

从 5 个面板扩展到 7 个：

| 序号 | 面板 | 图标 | 布局类型 |
|------|------|------|---------|
| 1 | 书籍信息 | `ReadOutlined` | A: 单列表单 |
| 2 | 世界规则 | `LayersOutlined` | B: 列表 + 编辑器 |
| 3 | 设定库 | `FileTextOutlined` | B: 分类列表 + 编辑器 |
| 4 | 标签库 | `TagsOutlined` | B: 树形分类 + 详情 |
| 5 | 创作区 | `EditOutlined` | C: 导航树 + 多视图 |
| 6 | 伏笔库 | `PushpinOutlined` | D: 搜索列表 |
| 7 | 正文库 | `BookOutlined` | D: 搜索列表 |

### 1. 书籍信息

**布局**：A 型（单列居中表单，max-width 720px）

**字段**（均为元数据，不耦合剧情，供 AI 生成参考）：

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| 书名 | * | Input | — |
| 题材 | * | Select | 修真仙侠、都市、科幻等 |
| 子题材 | * | Select | 凡人流、系统流等 |
| 发布平台 | * | Select | 起点中文网、番茄等 |
| 目标受众 | * | Select | 男频/女频 + 年龄段 |
| 标签 | — | TagSelect | 从标签库选取，多选 |
| 文笔文风 | * | Select | 细腻写实、轻松搞笑等 |
| 每章字数 | * | InputNumber | 预计每章字数 |
| 总字数 | — | InputNumber | 预计总字数 |
| 参考作品 | — | Input | 逗号分隔 |
| 核心卖点 | — | Input | 影响 AI 生成重心 |
| 简介 | * | TextArea | 300 字限制，影响 AI 核心参考 |

### 2. 世界规则

**布局**：B 型（列表 + 编辑器）

**规则结构**（平铺，无文件夹）：

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| 规则名称 | * | Input | — |
| 规则等级 | * | Select | 核心（绿）/ 重要（橙）/ 一般（灰） |
| 规则内容 | * | TextArea | — |

**状态**：空状态 → 列表（规则名称 + 等级 Tag）→ 新建/编辑表单

### 3. 设定库

**布局**：B 型（分类列表 + 编辑器/查看器）

**固定分类**：人物 | 物品 | 地点 | 势力 | 其他

**人物字段**：

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| 角色名称 | * | Input | — |
| 性别 | — | Select | 男/女/其他 |
| 性格 | — | Input | — |
| 特点/癖好 | — | Input | 特殊癖好 |
| 标签 | — | TagSelect | — |
| 简介 | — | TextArea | — |

**编辑模式**：无状态区，底部有「标记废弃」按钮（虚线红色边框）
**查看模式**：纯文本展示 + 底部状态区（虚线框，系统自动维护，不可编辑）

状态区标签：活跃/不活跃、已出场 N 章、当前境界/等级、存活/死亡 等

**废弃机制**：标记为废弃后不再被引用，查看时显示删除线

### 4. 标签库

**布局**：B 型（左侧大类列表 + 右侧子类树形 + 标签详情）

**结构**：
- 标签大类（用户自定义）：种族、性格、势力类型、物品等级、文风等
- 子类支持无限层级细分：如 种族 → 人族 / 妖族 → 海妖 / 陆妖
- 标签详情：名称、父级（可选）、详情描述

### 5. 创作区

**布局**：C 型（280px 导航树 + 多视图编辑区）

**核心规则**：
- 全部为编辑模式，无观看模式
- 右上角固定「保存」按钮，手动保存
- AI 操作收拢在 `AI ▾` 下拉按钮中

#### 5.1 总纲编辑

| 字段 | 类型 | 说明 |
|------|------|------|
| 整体方向 | TextArea | 全书大方向 |
| 阶段划分 | 可增删改列表 | 每个阶段：序号 + 阶段名 + 阶段简介 |

#### 5.2 卷纲编辑

| 字段 | 类型 | 说明 |
|------|------|------|
| 卷名 | 标题显示 | 如「第一卷 · 初入仙门」 |
| 卷核心冲突 | TextArea | 本卷核心矛盾 |
| 卷梗概 | TextArea | 本卷故事梗概 |
| 阶段划分 | 可增删改列表 | 同总纲阶段结构 |

#### 5.3 章纲编辑

**字段排序**（重要字段前置）：

| 优先级 | 字段 | 必填 | 类型 | 说明 |
|--------|------|------|------|------|
| 1 | 章节标题 | * | Input | — |
| 1 | 目标字数 | * | InputNumber | 放在第一排右侧 |
| 2 | 前章衔接 | — | TextArea | 与上一章的承接，辅助 AI |
| 2 | 下章悬念 | — | TextArea | 为下一章埋钩子，辅助 AI |
| 3 | 情节摘要 | — | TextArea | 本章核心情节 |
| 折叠 | 更多细节 | — | Collapse | 折叠次要信息，点击展开 |
| — | 场景 | — | Array Input | + 添加按钮 |
| — | 时间 | — | Input | — |
| — | 情绪基调 | — | Input | — |
| — | 出场人物 | — | TagSelect | 从设定库选取 |
| — | 重大事件 | — | Array Input | + 添加按钮 |
| — | 伏笔 | — | Array + 从伏笔库关联 | + 添加按钮 |
| — | 预计看点 | — | Input | — |
| 4 | 补充说明 | — | TextArea | — |

**状态**：由系统自动维护（草稿/已生成正文/已过审），不可用户编辑

#### 5.4 正文编辑

- Tab 切换：`章纲` ↔ `正文`
- **右上角按钮组**：`AI ▾` 下拉 + `保存` + `过审`（primary 填充）
- 章纲修改提示条：当章纲被修改后显示警告，提供「忽略」和「重新生成」按钮
- 底部状态栏：字数统计 / 保存状态

### 6. 伏笔库

**布局**：D 型（搜索 + 筛选 + 列表）

**筛选**：全部 / 未揭晓 / 已揭晓

**排序规则**：
- 未揭晓的优先显示
- 同状态内按埋下章节排序（越早越靠前）
- 已揭晓的条目降低不透明度（`opacity: 0.7`）+ 标题删除线

**条目结构**：
```
[伏笔名称] [状态Tag]          [第N章 →]
[伏笔描述]
```

### 7. 正文库

**布局**：D 型（列表）

**条目结构**：
```
[第N章 · 章名]                    [字数]
[正文预览（2行截断）]
[AI ▾]  [查看 →]
```

- 字数显示为灰色辅助文字，不使用 Tag
- AI 下拉按钮收拢所有 AI 操作（去AI味、润色、扩写）
- 查看链接跳转到正文全文

## 技术实现要点

### CSS 变量体系

在 `app/globals.css` 中维护语义化 CSS 变量（已在 v3 建立）：

```css
/* 核心色 */
--color-primary: #2F5D50;      /* 墨绿 */
--color-accent: #964400;        /* 朱砂 */
--color-danger: #ba1a1a;

/* 背景 */
--bg-page: #f5f1e8;            /* 宣纸 */
--bg-section: #f8f6f0;
--bg-card: #ffffff;

/* 文字 */
--text-primary: #181c22;
--text-secondary: #414753;
--text-muted: #717785;

/* 边框 */
--border: #e0e2eb;
--border-section: #ebedf7;
```

### 组件结构

- 使用 CSS Modules（`index.module.css`）+ camelCase 命名
- Ant Design v6.5.0 + `@ant-design/nextjs-registry`
- 面包屑：`antd Breadcrumb` + `@ant-design/icons` Outlined
- AI 下拉：`antd Dropdown` + `Menu`
- 确认弹窗：`antd Modal.confirm`
- 折叠区：`antd Collapse` 或自定义折叠组件
- 空状态：统一的 `EmptyState` 组件（icon + text + action）

### 文件路径

- 全局样式：`app/globals.css`
- 工作区布局：`app/pages/books/index.tsx` + `index.module.css`
- 面板配置：`app/pages/books/config/workspace-panels.tsx`
- 各面板组件：`app/pages/books/components/{panel-name}/`
- 共享组件：`shared/ui/`（如 AiDropdown、EmptyState、BreadcrumbNav）

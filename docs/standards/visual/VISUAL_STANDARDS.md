# 视觉标准规范（东方水墨风格）

> **版本**: v1.0
> **更新时间**: 2026-07-05
> **适用范围**: 全项目 UI 开发

本文档是项目视觉实现的唯一权威参考。所有新开发和修改的 UI 必须遵循以下规范，以确保「暖纸·东方水墨」风格的一致性。

---

## 1. 色彩规范

### 1.1 设计 Token 与语义映射

项目使用 CSS 变量定义色彩体系，**禁止**在组件中硬编码颜色值。

| 语义 | 变量名 | 色值 | 用途 |
|------|--------|------|------|
| 主色（青黛） | `--color-primary` | `#2F5D50` | 品牌标识、重要操作、高优先级内容 |
| 主色悬浮 | `--color-primary-hover` | `#1e4438` | 主色按钮 hover 状态 |
| 主色浅底 | `--color-primary-bg` | `rgba(47,93,80,0.05)` | 选中态背景 |
| 主色浅底悬浮 | `--color-primary-bg-hover` | `rgba(47,93,80,0.10)` | 悬浮背景 |
| 主色边框 | `--color-primary-border` | `rgba(47,93,80,0.20)` | 选中态边框 |
| 成功 | `--color-success` | `#2E8B57` | 成功状态、正面标签 |
| 警告 | `--color-warning` | `#964400` | 警告状态、重要标签 |
| 错误 | `--color-error` | `#ba1a1a` | 错误状态、危险操作 |

### 1.2 背景层级

| 层级 | 变量名 | 色值 | 使用场景 |
|------|--------|------|----------|
| 页面背景 | `--bg-page` | `#f0ece4` | 最底层页面底色 |
| 抬升面 | `--bg-elevated` | `#faf8f4` | 卡片、弹窗、顶栏、底栏 |
| 静音面 | `--bg-muted` | `#f5f2ec` | 折叠区、代码块、次要容器 |
| 强调底 | `--bg-strong` | `#e8e4dc` | 分割线背景、hover 态、进度条底色 |

### 1.3 文字层级

| 层级 | 变量名 | 色值 | 使用场景 |
|------|--------|------|----------|
| 主文字 | `--text-primary` | `#1a1814` | 标题、正文、重要信息 |
| 次文字 | `--text-secondary` | `#4a4640` | 副标题、说明文字 |
| 三级文字 | `--text-tertiary` | `#807b74` | 时间戳、placeholder、辅助信息 |
| 浅色文字 | `--text-light` | `#b5b0a8` | 禁用态、最弱层级 |
| 反白文字 | `--text-inverse` | `#ffffff` | 深色背景上的文字（primary 按钮内文字） |

### 1.4 边框色

| 层级 | 变量名 | 色值 | 使用场景 |
|------|--------|------|----------|
| 默认边框 | `--border` | `#ddd8d0` | 卡片、输入框、分隔线 |
| 强调边框 | `--border-strong` | `#c0bbb3` | 选中态、hover 边框 |
| 弱化边框 | `--border-light` | `#ebe7e0` | 分割线背景、极弱分隔 |

### 1.5 使用规则

1. **主色统一原则**：所有重要/高优先级内容（主操作按钮、选中态、核心标签）统一使用 `var(--color-primary)` 或其衍生变量
2. **功能色限定场景**：
   - 绿色（`--color-success`）仅用于成功状态、正面标签、已完成标记
   - 橙色（`--color-warning`）仅用于警告状态、重要标签
   - 红色（`--color-error`）仅用于错误状态、删除/危险操作
3. **禁止项**：
   - 禁止在组件中使用硬编码颜色值（如 `color: #2F5D50`），必须使用 CSS 变量
   - 禁止随意引入非规范色彩（如蓝色 `#1890ff`、紫色等）
   - 禁止在不同功能语义间混用颜色（如用绿色表示删除）
4. **传统色补充**：项目还定义了 `--color-jade`（青黛）、`--color-gold`（金）、`--color-indigo`（靛蓝）三个传统色，仅用于装饰性元素或特殊内容高亮

## 2. 字体规范

### 2.1 字体族定义

| 用途 | 变量名 | 字体栈 | 使用场景 |
|------|--------|--------|----------|
| 展示字体 | `--font-display` | `'Noto Serif SC', serif` | 页面标题、品牌名、卷纲标题等大号衬线文字 |
| 正文字体 | `--font-body` | `'Inter', 'Noto Sans', 'Noto Sans SC', sans-serif` | 正文内容、表单标签、按钮文字等所有 UI 文字 |
| 等宽字体 | `--font-mono` | `'JetBrains Mono', 'Fira Code', 'Consolas', monospace` | 代码、JSON、API 地址等等宽内容 |

### 2.2 字号与行高

| 层级 | 字号 | 行高 | 使用场景 |
|------|------|------|----------|
| 页面标题 | 18px | 1.4 | 顶栏品牌名、页面大标题 |
| 区域标题 | 16px | 1.5 | 面板标题、卡片标题 |
| 正文 | 14px | 1.6 | 所有正文内容、表单标签 |
| 辅助文字 | 13px | 1.5 | 说明文字、提示信息 |
| 小号文字 | 12px | 1.4 | 时间戳、统计数字、底部栏 |
| 最小文字 | 11px | 1.3 | 移动端底部栏、极小提示 |

### 2.3 使用规则

1. **标题一律使用 `--font-display`**（宋体衬线体），营造东方书卷气
2. **正文和所有 UI 文字使用 `--font-body`**，确保屏幕可读性
3. **代码块和等宽内容使用 `--font-mono`**，如 JSON 配置、API 路径
4. **禁止**在 CSS Modules 中直接写死字体名，必须通过变量引用
5. **font-weight** 规范：`600` 用于标题，`400` 用于正文，`500` 用于按钮和强调

## 3. 间距规范

### 3.1 间距阶梯

项目定义了 8 级间距变量，所有间距必须使用这些变量：

| 变量名 | 值 | 使用场景 |
|--------|-----|----------|
| `--space-1` | 4px | 图标与文字间距、紧凑元素微调 |
| `--space-2` | 8px | 按钮组内部间距、Tag 之间间距 |
| `--space-3` | 12px | 表单项之间间距、列表项间距 |
| `--space-4` | 16px | 区块内边距、标题与内容间距 |
| `--space-5` | 20px | 面板内边距（中等） |
| `--space-6` | 24px | 弹窗内边距、顶栏左右内边距 |
| `--space-8` | 32px | 大区块间距、页面上下内边距 |

### 3.2 使用规则

1. **小元素间距**（图标+文字、Tag 之间）：`space-1` ~ `space-2`
2. **组件内部间距**（表单项之间、列表项之间）：`space-3` ~ `space-4`
3. **区块间距**（面板之间、区域之间）：`space-5` ~ `space-6`
4. **页面级间距**（内容区上下边距）：`space-8`
5. **禁止**使用任意数值（如 `13px`、`17px`），必须取最近的阶梯值
6. **间距使用 CSS 变量**：`gap: var(--space-2)` 而非 `gap: 8px`

## 4. 圆角规范

### 4.1 圆角阶梯

| 变量名 | 值 | 使用场景 |
|--------|-----|----------|
| `--radius-sm` | 0.25rem (4px) | 小型元素：Tag、Badge、小按钮 |
| `--radius-md` | 0.375rem (6px) | 输入框、选择器、顶栏图标按钮 |
| `--radius-lg` | 0.5rem (8px) | 卡片、弹窗、代码块 |
| `--radius-xl` | 0.75rem (12px) | 大卡片、面板容器 |
| `--radius-full` | 9999px | 圆形按钮（`shape="circle"`）、圆形头像、药丸形元素 |

### 4.2 使用规则

1. **按钮**：圆形按钮使用 `--radius-full`，方形按钮不设圆角（Ant Design 默认）
2. **卡片/容器**：统一使用 `--radius-lg` 或 `--radius-xl`
3. **输入框/选择器**：统一使用 `--radius-md`
4. **Tag/Badge**：统一使用 `--radius-sm`
5. **禁止**在组件中直接写死圆角数值

## 5. 阴影规范

### 5.1 阴影层级

| 变量名 | 定义 | 使用场景 |
|--------|------|----------|
| `--shadow` | `0 1px 3px rgba(24,28,34,0.08)` | 默认阴影：卡片、下拉菜单、顶栏 |
| `--shadow-md` | `0 4px 12px rgba(24,28,34,0.12)` | 中等阴影：弹窗（Modal）、悬浮面板 |
| `--shadow-lg` | `0 8px 24px rgba(24,28,34,0.16)` | 重阴影：全屏弹窗、下拉层级最深 |

### 5.2 使用规则

1. **默认卡片/面板**：使用 `--shadow`
2. **弹窗/Modal**：使用 `--shadow-lg`
3. **下拉菜单/Popover**：使用 `--shadow-md`
4. **页面内容区不需要阴影**——项目使用暖纸色背景层级区分，而非阴影
5. **禁止**在组件中直接写死 `box-shadow` 数值，必须引用变量
6. 阴影色使用暖灰色调（`rgba(24,28,34,...)`），与暖纸色背景协调

## 6. 分割线规范

### 6.1 标准分割方式

项目**不使用**显式的 `<Divider>` 元素作为内容分割。统一采用 `gap + background` 的伪分割线方式：

```css
/* 标准分割方式 */
.separator {
  display: flex;
  flex-direction: column;
  gap: 1px;
  background: var(--border-light);
}

.separator > * {
  background: var(--bg-elevated);
}
```

### 6.2 使用规则

1. **面板/区域之间**：使用 `gap: 1px` + `background: var(--border-light)` 方式实现分割
2. **顶栏/底栏边框**：使用 `border-bottom: 1px solid var(--border)` 和 `border-top: 1px solid var(--border)`
3. **列表项之间**：使用 `gap: 1px` + `background: var(--border-light)` 或相邻元素间 `border-bottom`
4. **禁止**在内容区域使用 `<Divider />` 组件进行分割
5. **禁止**使用 `border-bottom` 或 `border-top` 在内容区域做分割（顶栏/底栏除外）

## 7. 按钮规范

### 7.1 按钮类型与使用场景

| 类型 | 适用场景 | 示例 |
|------|----------|------|
| `type="primary"` | 页面主操作、表单提交、确认操作 | 保存、创建、确认、测试连接 |
| `danger` | 删除操作、不可逆的危险操作 | 删除、移除、清空 |
| `type="text"` | 轻量辅助操作、工具栏按钮 | 更多操作（⋯）、展开/折叠、工具图标按钮 |
| `type="link"` | 行内文字链接式操作 | 新建大类、添加子标签 |
| 默认（无 type） | 次要操作、取消操作 | 取消、查看、关闭 |

### 7.2 使用规则

1. **每个页面/弹窗只允许一个 primary 按钮**，作为主操作
2. **删除操作统一使用 `danger` 按钮**，并在点击后弹出 `Modal.confirm` 二次确认
3. **`type="text"` 仅用于工具栏图标按钮**（如搜索、布局切换、更多操作），禁止混用于表单提交
4. **`type="link"` 仅用于行内文字式操作**（如列表头的「+ 新建」链接），禁止混用于主要操作
5. **取消操作统一使用默认按钮**（无 type），不使用 `type="text"`
6. **确认删除弹窗**统一使用 `confirmDelete` 工具函数（`shared/ui/confirm-delete`），文案格式为：`确定要删除「{名称}」吗？此操作不可撤销。`
7. **保存按钮**如需独立展示，使用 `shared/ui/save-button` 组件

### 7.3 按钮尺寸

| 场景 | 尺寸 |
|------|------|
| 弹窗操作按钮 | `size` 不设（默认 default） |
| 工具栏图标按钮 | `size="small"` |
| 行内操作按钮 | `size="small"` |
| 表单提交按钮 | `size` 不设（默认 default） |

## 8. 标签规范（Tag）

### 8.1 标签颜色映射

项目使用 Ant Design `Tag` 组件，并通过 CSS 变量定义了三个等级的标签颜色：

| 等级 | 含义 | Tag 颜色 | 背景变量 | 文字变量 | 使用场景 |
|------|------|----------|----------|----------|----------|
| 核心 | 最高优先级、核心内容 | `color="green"` | `--tag-core-bg` | `--tag-core-color` | 核心冲突、关键看点、已写正文 |
| 重要 | 需要关注、重要标记 | `color="red"` 或 `color="orange"` | `--tag-important-bg` | `--tag-important-color` | 卷纲标记、章节数量统计 |
| 一般 | 中性分类、辅助信息 | 默认（无 color） | `--tag-general-bg` | `--tag-general-color` | 书籍分类、普通状态、一般标签 |

### 8.2 使用规则

1. **核心标签**（绿色）：用于正面/完成状态、核心内容标记
   - 书籍题材分类：`<Tag color="green">{genre}</Tag>`
   - 已写正文标记：`<Tag color="green">已写正文</Tag>`
   - 成功状态：`<Tag color="green">连接成功</Tag>`
2. **重要标签**（红色/橙色）：用于需要关注的高亮信息
   - 卷纲标记：`<Tag color="red">卷纲</Tag>`
   - 章节数统计：`<Tag color="red">{count} 章</Tag>`
   - 只读标记：`<Tag color="orange">只读</Tag>`
3. **一般标签**（默认色）：用于中性分类和辅助信息
   - 发展弧线节点：`<Tag>{step}</Tag>`
   - 未保存状态：`<Tag>未保存</Tag>`
4. **禁止**随意引入其他颜色标签（如蓝色、紫色），必须遵循三级映射
5. Tag 尺寸统一使用默认尺寸，不使用 `size="small"`

## 9. 表单规范

### 9.1 Label 位置

- **垂直布局（表单页面）**：Label 位于输入框上方，使用 `<Form.Item label="...">`
- **水平布局（弹窗内）**：Label 位于输入框左侧，使用 `<Form.Item label="...">`
- **自定义表单**（非 Ant Design Form）：使用 `<label>` 标签，样式使用 `formLabel` CSS 类

### 9.2 必填标记

- 必填字段使用 Ant Design `<Form.Item required>` 或 `rules={[{ required: true }]}`
- 必填标记（红色星号）由 Ant Design 自动渲染，禁止手动添加 `*` 或「必填」文字
- 可选字段不显示任何标记

### 9.3 校验提示

- 校验错误信息通过 `<Form.Item rules>` 的 `message` 字段自定义
- 校验提示显示在输入框下方，使用 Ant Design 默认样式
- 禁止使用 `message.error()` 替代表单内联校验（仅用于 API 错误反馈）

### 9.4 输入框规范

| 类型 | 使用场景 |
|------|----------|
| `Input` | 短文本输入（书名、标签名等） |
| `Input.TextArea` | 长文本输入（简介、正文、描述等） |
| `Select` | 选项选择（题材、平台、下拉选择） |
| `InputNumber` | 数值输入 |
| `Slider` | 范围滑动选择 |

### 9.5 表单布局规则

1. 弹窗内表单：标题 → 表单项 → 底部操作栏（取消 + 提交）
2. 面板内表单：左侧字段区 + 右侧操作栏（标签库、AI 配置等面板）
3. 表单项之间间距：`gap: var(--space-4)` 或使用 `<Form.Item>` 自带间距
4. 表单操作栏靠右对齐，取消按钮在左、提交按钮在右

## 10. 空状态规范

### 10.1 EmptyState 组件

项目使用 `shared/ui/empty-state` 组件统一展示空状态：

```tsx
<EmptyState
  icon={<BookOutlined />}
  title="选择一个标签大类"
  description="从左侧选择大类查看其下标签，或新建大类开始"
  action={<button onClick={handleCreate}>+ 新建大类</button>}
/>
```

### 10.2 使用规则

1. **所有无数据场景必须使用 `EmptyState` 组件**，禁止使用纯文字或 `<Empty>` 组件
2. **图标**：使用 `@ant-design/icons` 中的 Outlined 风格图标，字号由组件控制
3. **标题文案**：简洁描述当前状态，不超过 10 个字
4. **描述文案**：提供操作引导，说明如何获取内容，不超过 25 个字
5. **操作按钮**：可选，提供下一步操作入口
6. **禁止**在空状态中使用自定义 SVG 图标或第三方图标

### 10.3 标准文案示例

| 场景 | 标题 | 描述 |
|------|------|------|
| 标签库未选择大类 | 选择一个标签大类 | 从左侧选择大类查看其下标签，或新建大类开始 |
| 正文库为空 | 还没有存稿 | 在创作区写完正文后可存入这里 |
| 搜索无结果 | 没有找到匹配的内容 | 尝试其他关键词或清除搜索条件 |
| 文件夹为空 | 选择文件开始编辑 | 从左侧文件树中选择一个文件 |

## 11. 弹窗规范（Modal）

### 11.1 Modal 尺寸

| 场景 | 宽度 | 使用场景 |
|------|------|----------|
| 小型弹窗 | 480px | 简单确认弹窗、少量字段编辑 |
| 中型弹窗 | 600px | 创建/编辑表单（默认尺寸） |
| 大型弹窗 | 720px | 内容预览、大段文本展示 |
| 全屏弹窗 | calc(100vw - 48px) | 特殊全屏编辑场景 |

### 11.2 标题格式

1. **弹窗标题**使用中文四字或五字短语，统一为「动词 + 名词」格式
2. 标准标题示例：
   - 创建新书
   - 编辑书籍
   - 确认删除
   - 编辑卷纲
3. 禁止使用英文标题
4. 禁止在标题中使用图标

### 11.3 Footer 布局

| 模式 | 使用场景 | 布局方式 |
|------|----------|----------|
| `footer={null}` | 自定义底部操作栏 | 在 `<form>` 内手动放置按钮 |
| `footer={[<Button />, ...]}` | 简单关闭按钮 | 使用数组传递按钮 |
| `Modal.confirm` | 删除确认等危险操作 | Ant Design 内置确认弹窗 |

### 11.4 Footer 按钮排列

- **取消按钮在左，确认按钮在右**
- 取消按钮：默认样式（无 type）
- 确认按钮：`type="primary"`
- 删除确认：使用 `Modal.confirm`，`okType="danger"`，`okText="删除"`

### 11.5 使用规则

1. **弹窗使用 `Modal` 组件**，禁止自定义弹窗实现
2. **确认删除操作统一使用 `Modal.confirm`** 或 `confirmDelete` 工具函数
3. 弹窗内表单使用 `footer={null}`，底部按钮通过自定义 `modalFooter` 样式类布局
4. 弹窗 `onCancel` 统一关闭弹窗并重置表单状态
5. 弹窗标题使用 `title` 属性，禁止在弹窗内容区重复显示标题

## 12. 响应式规范

### 12.1 断点定义

| 断点 | 条件 | 内容区内边距 | 适用场景 |
|------|------|-------------|----------|
| 桌面 | `> 1024px` | 160px 水平 | 大屏桌面 |
| 平板 | `768px ~ 1024px` | 48px 水平 | 小屏桌面、平板 |
| 移动端 | `< 768px` | 16px 水平 | 手机 |
| 极小屏 | `< 480px` | 16px 水平 | 小屏手机 |

### 12.2 移动端适配规则

1. **顶栏**：`padding` 缩减为 `0 12px`，副标题（`shell-subtitle`）隐藏
2. **底栏**：`padding` 缩减为 `0 12px`，右侧信息组隐藏
3. **品牌名**：字号从 18px 缩减为 14px（`< 480px` 时）
4. **内容区**：使用 CSS 变量 `--content-padding-x` 自动响应
5. **活动栏（Activity Bar）**：保持 56px 宽度不变

### 12.3 响应式规则

1. **断点使用 CSS Media Query**，在 `globals.css` 中统一定义
2. **禁止**在组件 CSS Modules 中定义响应式断点（全局响应式规则由 `globals.css` 管理）
3. **内容最大宽度**：`--content-max-width: 1024px`，居中显示
4. 弹窗宽度在移动端自动适配为 `calc(100vw - 32px)`
5. 禁止在移动端隐藏核心操作按钮

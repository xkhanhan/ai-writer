# 设置页活动栏改造 — 产品需求文档 (PRD)

> **版本：** v2.0  
> **状态：** 已评审  
> **最后更新：** 2026-07-11  
> **关联任务：** F-008  
> **视觉稿：** `docs/mockups/settings-activity-bar.html` (v6)  
> **必读对象：** 开发工程师（开发前）、代码审计员（测试前）

---

## 一、背景

当前设置页是两个 Tab 横向切换（AI 配置 / 提示词库），存在三个问题：
1. 与 workspace 视觉风格不一致（workspace 用活动栏，设置页用 Tab）
2. 无法返回上一页（只有返回首页按钮）
3. 扩展性差（后续加新功能无处放置）

---

## 二、术语表

| 术语 | 含义 |
|------|------|
| **活动栏** | 页面最左侧的窄条（56px），垂直排列图标按钮，用于切换内容区 |
| **配置列表** | AI 配置页的左侧列表（232px），展示所有 AI 配置条目 |
| **详情区** | AI 配置页的右侧区域，展示选中配置的完整信息 |
| **厂商联动** | 选择厂商后自动填充 Base URL、模型列表、上下文大小、温度范围 |

### 3.3 AI 配置页 — 列表 + 详情

#### 3.3.1 配置列表（左侧面板，232px）

**头部：**
- 左侧：标题"AI 配置"（`font-family: var(--font-display)`，14px）
- 右侧：`+` 按钮（`PlusOutlined`，jade 色），点击弹出创建 Modal

**列表项（每个配置条目）：**

```
┌──────────────────────────┐
│ OpenAI GPT-4o    [已连接] │  ← 名称 + 状态徽章
│ OpenAI · gpt-4o          │  ← 厂商 · 模型
│ sk-proj-****pqr6         │  ← API Key 掩码
└──────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 内边距 | 12px 16px |
| 选中态 | `background: var(--accent-soft); border-left: 2px solid var(--accent)` |
| 名称 | 13px, font-weight 500 |
| 状态徽章 | 9px，已连接=jade 色背景，未测试=灰色边框 |
| API Key 掩码 | 等宽字体 11px，显示前 7 位 + `****` |

#### 3.3.2 详情区（右侧，flex:1）

**顶部工具栏：**

```
┌────────────────────────────────────────────────────┐
│ OpenAI GPT-4o  [已连接]           [编辑] [删除]      │
└────────────────────────────────────────────────────┘
```

- 配置名称：`var(--font-display)`，15px，font-weight 700
- 状态 Tag：`antd Tag`，success/default
- 操作按钮：编辑（`EditOutlined`）、删除（`DeleteOutlined`, danger）

**详情网格（2列）：**

```
┌─────────────────┬─────────────────┐
│ 厂商             │ API 格式         │
│ OpenAI           │ OpenAI Compatible│
├─────────────────┴─────────────────┤
│ Base URL                          │
│ https://api.openai.com/v1         │  ← 等宽字体
├───────────────────────────────────┤
│ API Key                           │
│ sk-proj-****pqr6  👁              │  ← 掩码 + 眼睛切换
├─────────────────┬─────────────────┤
│ 模型             │ 状态             │
│ gpt-4o          │ 已连接 [✓]       │
├─────────────────┼─────────────────┤
│ 上下文大小        │ 温度             │
│ 128K            │ 0.7             │
└─────────────────┴─────────────────┘
```

**API Key 掩码逻辑：**
- 默认显示：`apiKey.substring(0,7) + '****' + apiKey.substring(apiKey.length-4)`
- 示例：`sk-proj-abc123****vwx234`
- 点击眼睛图标（`EyeOutlined` ↔ `EyeInvisibleOutlined`）切换明文/掩码

**底部操作：**

```
────────────────────────────────────
[测试连接]  验证 API Key 和模型是否可用
```

点击"测试连接"按钮 → 调用测试 API → 结果用 `Alert` 组件展示（success/error/info + loading）。

#### 3.3.3 创建/编辑 Modal

**Modal 属性：**
- 标题：创建="新建 AI 配置" / 编辑="编辑 {配置名称}"
- 宽度：560px
- `destroyOnClose: true`

**表单字段：**

| # | 字段 | 组件 | 必填 | 默认值 | 联动 |
|---|------|------|------|--------|------|
| 1 | 配置名称 | `Input` | 是 | 创建空 / 编辑预填 | 无 |
| 2 | 厂商选择 | `Select` | 是 | 创建空 / 编辑预填 | **联动字段 3-7** |
| 3 | API 格式 | `Select` | 是 | 跟随厂商 | 随厂商切换 |
| 4 | Base URL | `Input` | 是 | 跟随厂商 | 随厂商切换 |
| 5 | API Key | `Input.Password` | 是 | 创建空 / 编辑预填 | 无 |
| 6 | 模型 | `Select` + 搜索 | 是 | 创建空 / 编辑预填 | 旁有"拉取模型"按钮 |
| 7 | 上下文大小 | `InputNumber` (K/M格式) | 否 | 跟随厂商默认值 | 随厂商切换 |
| 8 | 温度 | `Slider` + `InputNumber` | 否 | 跟随厂商温度范围 | 随厂商切换 |

**厂商联动映射表：**

| 厂商 | API 格式 | Base URL | 默认模型 | 默认上下文 | 温度范围 |
|------|---------|----------|---------|-----------|---------|
| OpenAI | openai | https://api.openai.com/v1 | gpt-4o | 128000 | 0-2 |
| Anthropic | anthropic | https://api.anthropic.com | claude-sonnet-4-20250514 | 200000 | 0-1 |
| DeepSeek | openai | https://api.deepseek.com | deepseek-chat | 64000 | 0-2 |
| 自定义 | custom | （空） | （空） | 128000 | 0-2 |

**模型拉取按钮：**
- 位置：模型 Select 旁
- 前置条件：API Key 和 Base URL 均已填写
- 行为：调用 `/api/ai/models` → 返回模型列表 → 填充 Select 选项
- Loading 状态：按钮显示 Loading 图标

**Modal 底部（Footer）：**

```
┌──────────────────────────────────────────────────────┐
│ [取消]                     [测试连接] [创建并保存]       │
│                              ↑ 已验证/验证失败 Tag      │
└──────────────────────────────────────────────────────┘
```

#### 3.3.4 自动测试连接

**触发时机：**
1. 用户点击 Modal 底部"创建并保存"或"保存"按钮时
2. 用户点击详情区"测试连接"按钮时

**流程：**

```
用户点保存
    ↓
自动调用测试 API（POST /api/ai/test）
    ↓
┌─ 测试中 → Modal 底部显示 Loading，保存按钮禁用
│
├─ 测试成功 → 显示成功 Tag [已验证] → 执行保存 → 关闭 Modal
│
└─ 测试失败 → 显示错误 Alert（含错误信息）→ 阻止保存 → 用户可修改后重试
```

**错误处理：**
- 网络错误：Alert 显示"连接超时，请检查网络"
- 认证失败：Alert 显示"API Key 无效"
- 模型不存在：Alert 显示"模型 xxx 不可用"
- 其他错误：Alert 显示服务端返回的错误信息

### 3.4 页面切换

| 活动项 | 内容区展示 |
|--------|----------|
| AI 配置 | 配置列表 + 详情区 |
| 提示词库 | 三栏布局（见 `prompt-library-redesign.md`） |
| 未来扩展 | 预留空状态 |

**切换行为：**
- 点击活动项 → 内容区整体替换（不保留滚动位置）
- AI 配置切到提示词库 → 列表+详情区消失，三栏布局出现
- 切换时不自动保存任何未保存数据

## CRITICAL (1)

### CSS-001: 未定义变量 `--bg-base` 导致主题失效

**文件：**
- `app/pages/books/components/creation-zone/components/ai-result-panel/index.module.css:107`
- `app/pages/books/components/review-result-panel/index.module.css:209`

**问题：** `var(--bg-base)` 在 `globals.css` 中未定义，所有主题下 fallback 为透明/无背景，调试区域背景色不显示。

**修复：** 替换为 `var(--bg-muted)` 或 `var(--bg-elevated)`。

**验证：** grep 确认无 `--bg-base` 残留；深色主题下调试区域有背景色。

## HIGH (6)

### CSS-002: 内联样式泛滥 — 6 个组件无 CSS Module

| 组件 | 内联 style 数量 | 有 CSS Module? |
|------|----------------|---------------|
| `creation-zone/components/chapter-view/index.tsx` | 20+ | **无** |
| `creation-zone/components/volume-view/index.tsx` | 18+ | **无** |
| `creation-zone/components/outline-editor/index.tsx` | 10+ | **无** |
| `creation-zone/components/volume-form/index.tsx` | 12+ | **无** |
| `folder-file-editor/components/markdown-editor/index.tsx` | 8+ | **无** |
| `archive-view/index.tsx` | 5+ | 有但未使用 |

**问题：** 80+ 处内联样式，完全绕过 CSS Modules 和主题系统。深色主题下可能出现颜色异常。

**修复：** 为每个组件创建 `index.module.css`，将布局/颜色样式迁移为 CSS 类。仅保留真正动态的值（如条件 `display`）为内联。

**验证：** grep -r "style={{" 检查这些文件；深色主题下视觉正常。

### CSS-003: `!important` 违规 (tag-selector)

**文件：** `shared/ui/tag-selector/index.module.css:35-36`

```css
border-radius: var(--radius-lg) !important;
box-shadow: 0 6px 16px rgba(0, 0, 0, 0.12) !important;
```

**问题：** 违反 visual.md 规则 V-2。且 box-shadow 使用硬编码 rgba，深色主题下对比度不足。

**修复：** 用更具体的选择器或 ConfigProvider token 替代 !important；box-shadow 改用 `var(--shadow-md)`。

### CSS-004: `:global(.ant-xxx)` 违规 (9 处)

| 文件 | 行号 | 目标 |
|------|------|------|
| `tag-selector/index.module.css` | 40,45,50,54,59 | ant-select-tree-* |
| `ai-scene-modal/index.module.css` | 12,16 | ant-input-textarea-show-count |
| `book-info-form/index.module.css` | 61,67 | ant-btn |

**问题：** 违反 visual.md 规则 V-3，绕过 ConfigProvider 直接修改 antd 内部样式。

**修复：** tag-selector 的 TreeSelect 样式通过 ConfigProvider theme.components.TreeSelect 覆盖。ai-scene-modal 和 book-info-form 的影响较小，可后续处理。

### CSS-005: 硬编码 rgba 在 CSS 文件中

| 文件 | 行号 | 值 |
|------|------|-----|
| `review-result-panel/index.module.css` | 169 | `rgba(46, 139, 87, 0.2)` |
| `review-section/index.module.css` | 75 | `rgba(46, 139, 87, 0.2)` |
| `tag-selector/index.module.css` | 36 | `rgba(0, 0, 0, 0.12)` |
| `home/index.module.css` | 143 | `rgba(0, 0, 0, 0.06)` |

**问题：** 不随主题变化。深色主题下 rgba(0,0,0,*) 几乎不可见；成功色 border 在深色主题下颜色不匹配。

**修复：** 定义 `--color-success-border`、`--shadow-md` 等变量替代。

### CSS-006: 硬编码颜色在 TSX 中

| 文件 | 行号 | 值 |
|------|------|-----|
| `dashboard-charts.tsx` | 136-138 | `rgba(47,93,80, 0.15/0.35/0.6)` |
| `home/index.tsx` | 249 | `strokeColor="#d9d9d9"` |

**问题：** 内联硬编码颜色，不随主题变化。

**修复：** 定义 `--color-primary-15/35/60` 变量；Progress 的 strokeColor 使用 CSS 变量。

### CSS-007: 导航树 error 色 fallback 不一致

**文件：** `navigation-tree/index.module.css:83,110,113,119`

```css
color: var(--color-error, #ff4d4f);  /* antd 红 */
```

**问题：** fallback 值 `#ff4d4f` 是 antd 的红，不是项目的 `#ba1a1a`。如果变量加载失败会显示错误的颜色。

**修复：** 改为 `var(--color-error, #ba1a1a)`。

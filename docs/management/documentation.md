# 文档规范

> 项目文档的编写标准、目录结构、维护周期。

## 1. 目录结构

```
docs/
├── README.md              ← 全局索引（AI 第一入口）
├── frontend/              ← 前端规范（7 个子分类）
├── backend/               ← 后端规范（4 个子分类）
├── management/            ← 项目管理规范
├── plans/                 ← 功能设计文档
├── mockups/               ← 原型页面（HTML）
└── archive/               ← 历史归档（只读）
```

## 2. 编写原则

- **简约放置**：重要内容放在 AI 能第一时间看到的位置（README.md、各文档顶部摘要）。
- **详细链接**：详细内容通过链接引用，避免重复。
- **条款化**：规范文档使用"禁止/必须/建议"等明确措辞，不使用"尽量/可能"。
- **示例驱动**：关键规则附带 ✅/❌ 代码示例。

## 3. 文档分类

| 类型 | 目录 | 说明 |
|------|------|------|
| 规范文档 | `frontend/` `backend/` `management/` | AI 直接遵循的条款化规范 |
| 设计文档 | `plans/` | 功能设计方案、实施计划 |
| 原型页面 | `mockups/` | HTML 交互原型 |
| 历史归档 | `archive/` | 已完成的 PRD 和设计稿（只读） |

## 4. 维护规则

- 规范文档随代码变更同步更新。
- 新增规范条款必须在 README.md 索引中体现。
- 归档文档禁止修改，仅作为历史参考。

版本 v8 · 精简版 · 最后更新 2026-07-12

设计定位

创作区是 AI 辅助小说创作系统的核心工作台。采用「章纲驱动」的交互方向——章纲是主编辑区，正文仅供阅读回顾，总纲和卷纲作为蓝图参考始终可见。右侧 AI 面板预留空位，后续单独设计。

总纲（方向） → 卷纲（本卷） → 章纲（滚动规划） → 正文（实时写作）

蓝色节点 = 创作者主动编辑的内容 · 灰色节点 = AI 生成后供阅读

标签添加流程

  1. 点击「+」按钮 → 在标签列表末尾出现一个内联输入框（auto-focus）
  2. 输入文字 → 实时过滤关联库中的匹配项（人物标签过滤设定库，伏笔标签过滤伏笔库）
  3. 选择或回车 → 从下拉列表选择已有项（关联 ID），或直接回车创建新标签（自由文本）
  4. 确认添加 → 输入框消失，新标签出现在列表末尾，立即持久化到数据库
  5. ESC 或失焦 → 取消添加，输入框消失

标签删除流程

  1. Hover 标签 → 标签背景变深（--bg-page），右侧出现 × 图标
  2. 点击 × → 标签立即移除（乐观更新），后台异步删除关联
  3. 关联影响：人物标签删除仅移除章纲关联，不删除设定库中的角色。伏笔标签同理。

人物标签特殊行为

  • 人物标签关联设定库角色 ID（characterId），非自由文本
  • 点击人物标签 → 弹出角色详情抽屉（显示角色设定）
  • 设定库修改角色名字后，章纲中的人物标签实时同步更新显示名
  • 如果设定库删除了角色，章纲标签显示为灰色 + 删除线，提示「角色已删除」

伏笔标签特殊行为

  • 伏笔标签关联伏笔库 ID（foreshadowingId）
  • 标签右侧显示伏笔状态小圆点： 已揭示 · 隐藏中 · 已过期
  • 点击伏笔标签 → 弹出伏笔详情抽屉

10. 总纲与卷纲的编辑

10.1 总纲编辑

核心卖点多行文本吸引读者的关键元素
总纲 → AI 上下文：总纲内容会自动注入到所有 AI 请求的 context 中，确保 AI 理解全书方向，不会生成偏离主线的内容。

10.2 卷纲编辑

导航树点击卷标题 → 中心区切换到卷纲编辑视图

卷纲编辑视图包含以下字段：

字段类型说明

核心冲突多行文本本卷的核心矛盾（蓝图关键字段）

预计看点多行文本吸引读者继续阅读的钩子

10.3 卷纲如何约束章纲

约束机制：卷纲是蓝图，章纲是蓝图的具体执行。AI 在生成章纲时，会自动将卷纲的「核心冲突」「发展弧线」「重要节点」作为约束条件注入 prompt。章纲不应偏离卷纲设定的方向。
约束链路
卷纲 coreConflict + 卷纲 developmentArc + 卷纲 keyPoints → 注入 AI prompt → 生成章纲
  • AI 生成章纲时，程序自动从卷纲提取蓝图字段，组装为约束条件
  • 蓝图参考条始终可见，创作者写章纲时可随时参照
  • 如果章纲需要偏离卷纲 → 应先修改卷纲，再重新生成/调整章纲
  • 卷纲修改后，已存在的章纲不会自动更新（需用户手动触发重新规划）

12.2 完整创作流程

E 组件架构总览 四种面板 × 统一接口 × 数据流
面板 × 模式对照
面板组件 模式 输出格式 多轮 典型场景 核心交互
AiExplorePanel explore options Yes 总纲方向探索、题材选择 多选项卡片 → 选择/编辑/采纳
AiPlanPanel plan structured Yes 卷纲规划、章纲规划、角色创建 字段卡片 → 逐字段采纳/全部采纳
AiDraftPanel draft text No 正文生成、扩写、润色 全文预览 → 整体采纳
AiCheckPanel check report No 一致性检查、角色弧线分析 问题列表 → 逐条查看/修复
统一接口
// 所有面板共享相同的 props 接口
<AiPanelBase
  adapterId: string            // 适配器 ID（决定面板行为）
  context: {
    bookId: string
    chapterId?: string
    volumeId?: string
  }
  onAdopt: (results[]) => void // 采纳回调
  onClose?: () => void         // 关闭回调
/>

// 数据流（单向）
调用层传入 context → API: POST /api/ai/execute
  → AdapterExecutor 串通
  → 返回 AdapterExecutionResult
  → 面板渲染结果
  → 用户交互（编辑/选择）
  → onAdopt(results) → 调用层处理写入
面板知道的
- 当前适配器 ID 和模式
- 要生成的字段 / 输出格式
- 有哪些提示词模板
- 用户输入了什么想法
- 用户编辑/采纳了哪些卡片
- 对话历史（多轮模式）
面板不知道的
- 字段保护/锁定逻辑
- 采纳后数据写入哪里
- 表单的校验规则
- 业务规则
- 数据持久化
- 版本控制

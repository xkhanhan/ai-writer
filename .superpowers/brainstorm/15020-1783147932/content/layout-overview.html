当前问题与改造方向

!

内联样式泛滥

创作区 7 个子组件（OutlineEditor、VolumeView、VolumeForm、ChapterView、ChapterForm、ContentEditor、MarkdownEditor）大量使用内联 style 而非 CSS Modules，导致样式难以统一维护。

!

设计令牌未完全落地

globals.css 定义了完整 token 系统，但 CSS Module 中大量使用硬编码像素值（如 padding: 48px）而非变量引用。两套变量名（语义化 + 别名）并存增加认知负担。

!

面板间布局不统一

各面板的标题区、内容区、操作栏的 padding/margin/字体大小缺乏统一约束。BookInfoForm 用 48px padding，FolderFileEditor 用 24px，CreationZone 用 20px。

# 第一阶段 PRD：首页、书籍入口与 AI 基础链路

## 1. 阶段目标

第一阶段只做一个最小可运行产品，用来验证本地前后端和 AI 调用链路。

核心目标：

- 前端能打开首页。
- 首页能展示已有书籍。
- 首页能创建新书。
- 首页能进入 AI 配置。
- 后端能保存和读取第一阶段基础数据。
- 后端能保存和读取 AI 配置。
- 后端能调用 OpenAI-compatible API。
- 前端能发送测试消息给 AI。
- 前端能展示 AI 返回内容或错误信息。

第一阶段不进入正式小说创作流程，也不做复杂状态系统。

## 2. 产品范围

### 2.1 包含内容

第一阶段包含：

- 首页。
- 书籍列表。
- 创建新书。
- AI 配置。
- AI 测试调用。
- 基础本地数据保存。

### 2.2 不包含内容

第一阶段不做：

- 书籍创作页面。
- 卷管理。
- 章节管理。
- 正文编辑器。
- 提示词页面。
- 工作流页面。
- 程序自动组装上下文。
- AI 写作流程。
- AI 事实提取。
- AI 质量检查。
- 验收流程。
- 状态合并。
- 版本回滚。
- 用户系统。
- 云端同步。

## 3. 页面需求

### 3.1 首页

首页是第一阶段主页面。

首页需要提供：

- 页面标题：AI 小说创作工具。
- 书籍列表区域。
- 创建新书按钮。
- AI 配置入口。
- AI 测试入口或测试区域。

首页目标：

- 用户能看到已有书籍。
- 用户能创建新书。
- 用户能进入 AI 配置。
- 用户能测试 AI 链路是否可用。

### 3.2 书籍列表

书籍列表展示已创建书籍。

每本书显示：

- 书名。
- 简介。
- 创建时间。
- 最后更新时间。

第一阶段点击书籍后，只显示占位提示：

```text
书籍创作页面将在第二阶段实现。
```

### 3.3 空状态

当没有任何书籍时，首页显示：

```text
还没有书籍，先创建一本新书。
```

空状态提供“创建新书”按钮。

### 3.4 创建新书

第一阶段推荐使用弹窗创建新书，避免引入额外页面复杂度。

字段：

- 书名，必填。
- 简介，选填。

校验：

- 书名不能为空。
- 书名长度不超过 60 个中文字符。
- 简介长度不超过 500 个中文字符。

提交成功后：

- 后端创建书籍记录。
- 首页刷新书籍列表。
- 新书出现在列表中。

### 3.5 AI 配置

AI 配置用于保存本地调用 AI 所需的信息。

字段：

- `AI_API_KEY`
- `AI_BASE_URL`
- `AI_MODEL`

默认建议：

```text
AI_BASE_URL = https://api.openai.com/v1
AI_MODEL = gpt-4o-mini
```

要求：

- 前端不能直接调用 AI 平台。
- 前端不能获取完整 API Key。
- 后端负责保存配置、读取配置和调用 AI。
- 读取配置时，只返回是否已配置 API Key。

### 3.6 AI 测试

AI 测试只用于验证基础链路，不属于正式创作功能。

测试区域需要包含：

- Prompt 输入框。
- 测试按钮。
- 请求中状态。
- 结果展示区域。
- 错误提示区域。

默认测试 prompt：

```text
帮我写一段测试文本。
```

测试链路：

```text
前端输入 prompt
  ↓
后端接口
  ↓
AI 平台
  ↓
后端接口
  ↓
前端展示
```

## 4. 后端接口

### 4.1 获取书籍列表

```http
GET /api/books
```

成功返回：

```json
{
  "success": true,
  "books": [
    {
      "id": "book_001",
      "title": "凡人修仙测试书",
      "description": "一本测试用小说",
      "createdAt": "2026-07-02T10:00:00.000Z",
      "updatedAt": "2026-07-02T10:00:00.000Z"
    }
  ]
}
```

### 4.2 创建新书

```http
POST /api/books
```

请求：

```json
{
  "title": "凡人修仙测试书",
  "description": "一本测试用小说"
}
```

成功返回：

```json
{
  "success": true,
  "book": {
    "id": "book_001",
    "title": "凡人修仙测试书",
    "description": "一本测试用小说",
    "createdAt": "2026-07-02T10:00:00.000Z",
    "updatedAt": "2026-07-02T10:00:00.000Z"
  }
}
```

失败返回：

```json
{
  "success": false,
  "error": "书名不能为空"
}
```

### 4.3 获取 AI 配置

```http
GET /api/ai/config
```

成功返回：

```json
{
  "success": true,
  "config": {
    "baseUrl": "https://api.openai.com/v1",
    "model": "gpt-4o-mini",
    "hasApiKey": true
  }
}
```

### 4.4 保存 AI 配置

```http
POST /api/ai/config
```

请求：

```json
{
  "apiKey": "sk-xxxx",
  "baseUrl": "https://api.openai.com/v1",
  "model": "gpt-4o-mini"
}
```

成功返回：

```json
{
  "success": true
}
```

失败返回：

```json
{
  "success": false,
  "error": "AI_API_KEY 不能为空"
}
```

### 4.5 AI 测试聊天

```http
POST /api/ai/chat
```

请求：

```json
{
  "prompt": "帮我写一段测试文本"
}
```

成功返回：

```json
{
  "success": true,
  "content": "AI 返回内容"
}
```

失败返回：

```json
{
  "success": false,
  "error": "错误信息"
}
```

## 5. 数据设计

第一阶段推荐使用本地 JSON 文件保存数据，暂不引入数据库。

建议数据目录：

```text
data/
  books.json
  ai-config.json
```

### 5.1 书籍数据

```json
{
  "books": [
    {
      "id": "book_001",
      "title": "凡人修仙测试书",
      "description": "一本测试用小说",
      "createdAt": "2026-07-02T10:00:00.000Z",
      "updatedAt": "2026-07-02T10:00:00.000Z"
    }
  ]
}
```

### 5.2 AI 配置数据

```json
{
  "apiKey": "sk-xxxx",
  "baseUrl": "https://api.openai.com/v1",
  "model": "gpt-4o-mini"
}
```

安全要求：

- `data/ai-config.json` 不提交 Git。
- API Key 只允许后端读取。
- 前端不能拿到完整 API Key。

## 6. 验收标准

第一阶段完成时，应满足：

- 本地前端页面可以打开。
- 首页能展示空状态。
- 首页能创建新书。
- 创建后首页能展示新书。
- 首页能进入 AI 配置。
- AI 配置能保存。
- AI 配置读取时不泄露完整 API Key。
- AI 测试能把 prompt 发给后端。
- 后端能调用 AI 平台。
- AI 返回内容能显示在前端。
- AI 调用失败时，前端能显示错误。
- 前端不直接持有或调用 API Key。
